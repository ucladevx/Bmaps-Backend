Events¶

Welcome to the Mappening Events API! Through a RESTful interface, this API provides information on all the events happening around UCLA. The simplest way to access this data is to go to the url whatsmappening.io/api/v1/events to get a comprehensive listing of all upcoming UCLA events. See the explanation of Event objects below. The Events API offers various ways to search and filter the event data and also allows for independent processing and filtering.

Event¶

An Event object is a GeoJSON with the following keys:

geometry¶

The geometry object contains the coordinates for the Event in the format required by Mapbox. The geometry data must be set according to the location information of the Event. If no location information is provided or can be supplied through the locations API, the event pin cannot be displayed with Mapbox.

Event Object Structure:¶
geometry: { type: "Point", coordinates: [ ``latitude``, ``longitude`` ]}
type¶

The type of the geometry object. Should always be of value "Point" as Event locations are treated as a point with a singular (latitude, longitude) pair.

Note

An geometry’s type will always be of value “Point”.

geometry.coordinates¶

The coordinates of the geometry object. Contains latitude and longitude Number values that correspond to the location of the Event.

Latitude:a Number value between the range (-90, 90)
Longitude:a Number value between the range (-180, 180)
id¶

The id contains a unique ID for the Event object.

Note

Each id is unique. This allows for searching for events by index.

properties¶

Contains all the Event information and will be explored further below.

properties Object Structure:¶
properties: {
  category: "<NONE>",
  duplicate_occurence: false,
  is_cancelled: false,
  name: "Mappening Launch",
  start_time: "2018-08-21T04:20:00-0700",
  stats: { attending: #, interested: #, maybe: #, noreply: # }
}

Mandatory Event Properties

These properties must have a valid value for every event.

category¶

The category of the Event. All the categories can be seen by dynamically calling /categories. About half of events have a category and the rest have <NONE>. TODO is this mandatory because I see events where it is missing.

name¶

The String of the Event’s name. A specific event can be searched by calling /name/<event_name>.

stats¶

The JSON for events from Facebook with attendance stats with a ~6 hour accuracy. Contains 4 keys: attending, noreply, interested, and maybe, each with an integer value.

start_time¶

The String start time of the Event in the format Sat, 17 Feb 2018 23:30:00 GMT-0800.

is_cancelled¶

The Boolean indicating that the Event is cancelled.

duplicate_occurrence¶

The Boolean indicating this is a single Event taking place on multiple days i.e a weekly farmers market, not a multiday event like a hackathon.

Potential Event Properties

If these details/properties aren’t present, the JSON keys won’t be present.

cover_picture¶

The String url to the cover picture associated with the Event.

time_updated¶

The time the Event was last updated.

description¶

The String of the Event’s description.

place¶

A JSON with a location key with a mandatory country, city, latitude, and longitude. Other potential place details such as name can be seen in the example event below.

hoster¶

The String of the host name.

ticket_uri¶

The link to the Event’s ticketing.

end_time¶

The String end time of the Event in the format Sat, 17 Feb 2018 23:30:00 GMT-0800.

free_food¶

Whether or not the Event has free food, currently just a strong NO.

API DOCS¶

mappening.api.events.get_all_events()¶
Route:/
Description:Returns a GeoJSON of all events within a few miles of UCLA
mappening.api.events.get_event_by_id(event_id)¶
Route:/id/<event_id>
Description:Returns GeoJSON of singular event matching event ID
Parameters:event_id (str) – ID string to search database for exact match
mappening.api.events.get_event_by_name(event_name)¶
Route:/name/<event_name>
Description:Returns GeoJSON of singular event matching event name
Parameters:event_name (str) – case-insensitive name string to search database for exact match
mappening.api.events.get_event_categories(event_date)¶
Route:/categories/<event_date>
Description:Returns JSON of all event categories used in all events. Can also find all event categories for events that start on a given date. Potential Categories: Crafts, Art, Causes, Comedy, Dance, Drinks, Film, Fitness, Food, Games, Gardening, Health, Home, Literature, Music, Other, Party, Religion, Shopping, Sports, Theater, Wellness Conference, Lecture, Neighborhood, Networking
Parameters:event_date (str or None) – An optional case-insensitive string with raw date format or a commonly parseable format (e.g. DD MONTH YYYY -> 22 January 2018)
mappening.api.events.search_events()¶
Route:

/search?term=str&date=str&category=str
Description:

Returns GeoJSON of all events filtered by date, search term, and/or category. The search term is case insensitive and searched for in the event name. Useful for a search bar.
Parameters:
  • term (str or None) – An optional query component/parameter for the search term to filter by
  • date (str or None) – A query component/parameter for the date to filter by. Case-insensitive string with raw date format or a commonly parseable format (e.g. DD MONTH YYYY -> 22 January 2018)
  • category (str or None) – A query component/parameter for the event category to filter by
