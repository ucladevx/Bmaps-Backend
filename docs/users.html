Users¶

Welcome to the Mappening Users API! The simplest way to access the API is to go to the url whatsmappening.io/api/v2/users. See the explanation of the API below. The Users API is NOT intended to be a public API, admin permissions required to access any other user data.

User¶

A User object is a JSON with the following keys:

account¶

The account object contains info related to the user’s account with Mappening. Unique identifiers and user status/access permissions are set in the account fields.

account Object Structure:¶
account: {
  id: "unique_int",
  is_active: true,
  is_admin: false,
  username: "str",
  password_hash: "str",
  time_joined: "datetime",
  time_updated: "datetime"
}
id

Uniquely identifies the User object. Should be a unique int, typically from Facebook accounts.

is_active¶

Specifies whether or not a User is active. If true, User has an activated account that they can log in to, otherwise account will be rejected/suspended from use. User information remains in the database if deactivated.

is_admin¶

Specifies whether or not a User has admin privileges. An admin can grant and remove privileges at will. Should probably create a superpowered admin but devs are too lazy.

username¶

TODO: implement username/password accounts rather than just Facebook Authentication.

password_hash¶

TODO: implement username/password accounts rather than just Facebook Authentication.

time_joined¶

The date and time in UTC that the User registered a User account with Mappening.

time_updated

The date and time in UTC that the User last logged in or their User info was last updated.

app¶

The app object contains the app-related information for the User, or information that gives each User a personalized experience on Mappening.

app Object Structure:¶
app: {
  filters: [`filter`, `filter`],
  favorites: [`event_id`, `event_id`],
  past_events: [`event_id`, `event_id`]
}
filter¶

Keeps track of what filters the User is currently using. If conflicting filters are present (such as now and upcoming), they are both present in the database but the Events filtering API will only return appropriate results for the first received filter.

Allowed values: ‘now’, ‘upcoming’, ‘period’, ‘morning’, ‘afternoon’, ‘night’, ‘oncampus’, ‘offcampus’, ‘nearby’, ‘popular’, ‘food’

favorites¶

The events that the User has favorited. Only the event ID is stored in the database and there is no guarantee that the event ID is a valid ID.

past_events¶

The events that the User has (TODO:) attended in the past. Only the event ID is stored in the database and there is no guarantee that the event ID is a valid ID.

personal_info¶

The personal_info object contains User personal information to help customize interaction with each User.

personal_info Object Structure:¶
personal_info: {
  email: "mappeningdevx@gmail.com",
  full_name: "Dora D. DevX",
  first_name: "Dora",
  last_name: "DevX"
}
email¶

The string email of the User. May not be present in which case the field will be an empty string. TODO: email validation.

full_name¶

The string full name of the User. Not limited to just first and last name. The value is not updated if the first and last names are updated.

first_name¶

The string first name of the User.

last_name¶

The string last name of the User.

Sample User Object¶
{
  "account": {
    "id": "1557193131009244",
    "is_active": true,
    "is_admin": true,
    "password_hash": "",
    "time_joined": "2018-05-12 12:26:03",
    "time_updated": "2018-05-13 07:08:49",
    "username": ""
  },
  "app": {
    "favorites": [
      "42",
      "69"
    ],
    "filters": [
      "now",
      "popular",
      "food"
    ],
    "past_events": [
      "101",
      "821"
    ]
  },
  "personal_info": {
    "email": "katrina@thewijayas.com",
    "first_name": "Katrina",
    "full_name": "Katrina Wijaya",
    "last_name": "Wijaya"
  }
}

mappening.api.users.search_users()¶
Route:GET /search?favorite=id
Description:Gets ID and name of users as filtered by a specified event ID. If user has specified event listed as a favorite then is included in results.
Parameters:favorite (str or None) – An optional query component/parameter where the ID of an event is passed
Returns:JSON of all users matching filters
Requires:Admin permissions
mappening.api.users.update_user(user_id)¶
Route:

PUT /<user_id>?active=false&admin=true&password=str&first_name=Katrina&last_name=Wijaya&email=mappeningdevx@gmail.com
Description:

Updates user with id user_id. Updates any optional fields that are set as query parameters.
Parameters:
  • user_id (int) – The int ID of a specific user
  • active (boolean or None) – An optional query component/parameter to update whether or not a user is active. If true, user has an activated account that they can log in to, otherwise account will be rejected/suspended from use
  • admin (boolean or None) – An optional query component/parameter to update whether or not a user has admin permissions. All admins have same permissions so maybe should create a super admin.
  • password (str or None) – An optional query component/parameter to update the password for a user. TODO: actually supporting passwords/salting/hashing.
  • first_name (str or None) – An optional query component/parameter to update the user’s first name. Does not modify full name stored in database.
  • last_name (str or None) – An optional query component/parameter to update the user’s last name. Does not modify full name stored in database.
  • email (str or None) – An optional query component/parameter to update the user’s email. TODO: email verification.
Returns:

JSON of updated user or an error message
Requires:

Admin permissions

  • active (boolean or None) – An optional query component/parameter to update whether or not a user is active. If true, user has an activated account that they can log in to, otherwise account will be rejected/suspended from use
  • admin (boolean or None) – An optional query component/parameter to update whether or not a user has admin permissions. All admins have same permissions so maybe should create a super admin.
  • password (str or None) – An optional query component/parameter to update the password for a user. TODO: actually supporting passwords/salting/hashing.
  • username (str or None) – An optional query component/parameter to update the username for a user. TODO: actually supporting usernames.
